## SOP-005-HANDOFF - Responder Handoff Standard

**Tags:** responder guidance, CAD note, dispatch summary, SBAR, radio update

Responder handoff must be structured, brief and continuously updated. MERS-AI should use an SBAR-style format.

Situation: What is happening now? Include incident type, priority and immediate danger.
Background: What led to the call? Include mechanism, symptom onset, caller description and number of patients.
Assessment: What the agent/dispatcher has identified from questions. Include conscious/breathing/bleeding status, hazards and uncertainty.
Recommendation/Response: Which responders are recommended, route/access notes, staging needs, hospital notification and caller instructions already given.

Example CAD/radio summary:
"P1 medical. Adult male collapsed at home, caller reports not breathing. Location: Block B, Level 9, Unit 09-12, near Masjid Al-Hidayah, lift available. Caller is daughter, phone number captured. CPR instructions started at 14:03. No known hazards. Recommend ALS ambulance; nearest access through guardhouse Gate 2."

Responder guidance should never include unverified diagnosis as fact. Use "suspected" when appropriate. Say "caller reports" for caller-provided information. Say "confirmed by dispatcher questions" only if the dispatcher has verified through structured questioning.

The AI should update responders whenever there is a priority-changing fact: breathing stops, bleeding becomes uncontrolled, fire spreads, suspect returns, victim location changes, caller evacuates, gate access changes, number of patients changes, or hazards are discovered.
