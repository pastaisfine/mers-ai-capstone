## SOP-003-MINIMUM-DATA - Minimum Data Set for Every Emergency Incident

**Tags:** data model, CAD, incident summary, RAG, structured extraction

The AI agent should extract and maintain a live incident record. The minimum data set should be stored even if the call ends early.

Required fields:
- incident_id: generated by system
- timestamp_opened
- caller_phone_number
- caller_name_if_available
- caller_language
- callback_possible: yes/no/unknown
- exact_location_text
- gps_coordinates_if_available
- access_notes: gate, floor, unit, landmark, route, traffic obstruction
- incident_category: medical, fire, police, rescue, disaster, hazmat, mental health, unknown
- incident_subtype
- priority_level: P1, P2, P3, P4 or MCI
- number_of_patients
- patient_age_and_sex_if_available
- conscious_status
- breathing_status
- severe_bleeding_status
- hazard_status
- caller_safety_status
- responder_types_recommended
- pre_arrival_instructions_given
- responder_updates_sent
- dispatcher_decision_status: AI recommendation pending/approved/overridden
- retrieved_sop_ids
- confidence_score
- escalation_reason_if_any

The agent must distinguish observed facts from caller assumptions. Example: "Caller says father collapsed and is not breathing" is stronger than "possible heart attack". In the responder handoff, use direct observable facts first, then suspected classification.

When data is unknown, the agent should write "unknown" rather than hallucinate. Unknown values are operationally useful because they tell responders what must still be verified.
